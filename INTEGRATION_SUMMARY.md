## Performance Considerations

### Caching Strategy
- Data loading cached with `@st.cache_data`
- Clustering computation regenerated on-demand
- Study logs reloaded on each save (to stay current)

### Optimization Tips
1. **For large datasets** (10,000+ students):
   - Use feature selection in Tab 2
   - Consider data sampling
   - Reduce number of clusters

2. **For frequent usage**:
   - Archive old study logs
   - Regular backend maintenance
   - Monitor CSV file sizes

### Load Times Expected
- Tab 1: 2-3 seconds (depends on dataset size)
- Tab 2: 5-10 seconds (depends on clustering algorithm)
- Tab 3: <1 second (pre-loaded data)
- Tab 4: <1 second (form submission)
- Tab 5: 2-3 seconds (analytics computation)
